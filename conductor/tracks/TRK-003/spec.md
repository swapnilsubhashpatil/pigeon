# Specification: Cascade Simulator

**Track ID:** TRK-003
**Type:** Feature
**Created:** 2026-05-13
**Status:** Draft

## Summary

BFS traversal of the shipment dependency graph from a disrupted node — identifies every downstream shipment, purchase order, and customer SLA affected, and computes total dollar exposure in real time.

## Context

A single delayed shipment can silently cascade into 10+ downstream failures. The Cascade Simulator traverses the in-memory dependency graph (loaded from TRK-001 seed data), propagates delays hop-by-hop, checks SLA deadlines for each affected shipment, and outputs a structured impact report with dollar exposure. This is the second stage of the Detection → Decision → Action pipeline.

## User Story

As a logistics manager, I want to see the full downstream impact of a disruption — which shipments are affected, which customers will face SLA breaches, and what the total dollar exposure is — so that I can prioritise my response.

## Acceptance Criteria

- [ ] `POST /api/v1/cascade/simulate` with `{ shipmentId, delayHours }` returns a `CascadeImpactReport`
- [ ] BFS traverses up to 5 hops from the trigger shipment using the in-memory graph
- [ ] Each downstream node includes `delay_hours`, `sla_breached`, `sla_exposure_usd`, `hop_depth`
- [ ] `total_sla_exposure_usd` is the sum of all breached SLA penalties
- [ ] `critical_path` is the chain from trigger to the highest-exposure downstream node
- [ ] `affected_customers` and `affected_purchase_orders` are de-duplicated arrays
- [ ] `GET /api/v1/cascade/graph` returns the full adjacency list for UI visualisation
- [ ] Disruption events stored in the in-memory store auto-trigger simulation and store the report

## Dependencies

- TRK-001 — dependency graph in store, `CascadeImpactReport` type, customer SLA penalty data
- TRK-002 — `store.addDisruption()` already triggers SSE broadcast; simulator attaches reports to disruptions

## Out of Scope

- Decision engine / reroute generation (TRK-004)
- Firestore / Cloud SQL graph storage — in-memory adjacency list only
- ML-based delay propagation likelihood — deterministic 85% attenuation per hop
