# Specification: Risk Engine

**Track ID:** TRK-002
**Type:** Feature
**Created:** 2026-05-13
**Status:** Draft

## Summary

Build the Predictive Risk Engine: ingest real signals from 5 external sources, compute per-leg risk scores (0–100) using a weighted formula, update the in-memory store every 15 minutes, and expose REST endpoints for current scores and manual refresh.

## Context

Pigeon's core value is detecting disruptions 24–72 hours early. The Risk Engine is the signal-to-score pipeline that makes this possible. It runs continuously, fusing weather, vessel, port, traffic, and geopolitical signals into a single weighted risk score per shipment leg, then rolls those up into a shipment-level score that drives the Command Center UI.

## User Story

As a logistics manager, I want live risk scores on every shipment updated every 15 minutes using real external signals, so that I can see which shipments are at risk before disruptions impact delivery.

## Acceptance Criteria

- [ ] Each signal source module returns a normalised 0–100 score (or falls back to a mock value if the API key is missing)
- [ ] Per-leg scores are computed from signals appropriate to the leg type (trucking / port / ocean)
- [ ] SLA urgency multiplier is computed live from `SLA_deadline` using current time
- [ ] `weighted_risk_score` stored in the in-memory store updates correctly after each refresh
- [ ] `POST /api/v1/risk-scores/refresh/:id` triggers a score recompute for a single shipment and returns updated scores
- [ ] `GET /api/v1/risk-scores` returns current scores for all 40 shipments
- [ ] Scheduler runs a full refresh every 15 minutes without crashing the server
- [ ] When `weighted_risk_score > 75`, a `DisruptionEvent` is added to the store and broadcast via SSE

## Technical Notes

- Signal modules live in `src/modules/risk-engine/signals/`
- Each signal module exports a single async function: `fetchXxxScore(params) → Promise<number>` (0–100)
- If API key missing or call fails: log warning, return mock score (pre-seeded realistic value)
- AIS stream: persistent WebSocket in `signals/vessel.ts`, caches last known position per vessel MMSI; position delta computed against expected position derived from shipment origin/destination and elapsed time
- Port coordinates: hardcoded map of UN/LOCODE → lat/lng in `signals/port.ts`
- Geopolitical: NewsAPI fetches headlines for affected region → Gemini `gemini-1.5-flash` classifies risk level (0=none, 1=low, 2=medium, 3=high) → maps to 0/15/35/60 score contribution
- Scorer weights per leg type:
  * Trucking: traffic 60%, geopolitical 40%
  * Port: port_congestion 70%, geopolitical 30%
  * Ocean: weather 50%, vessel_delta 30%, geopolitical 20%
