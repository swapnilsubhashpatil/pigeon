# Specification: Backend Foundation

**Track ID:** TRK-001
**Type:** Feature
**Created:** 2026-05-13
**Status:** Draft

## Summary

Scaffold the complete Pigeon backend — Express + TypeScript modular monolith with all shared data models, a typed in-memory store, and 40 synthetic shipments as seed data. Every other track builds directly on this foundation.

## Context

Pigeon is an autonomous supply chain co-pilot. The backend is a Node.js + Express monolith structured in three feature modules (risk-engine, cascade-simulator, decision-engine), each of which will become a separate track. This track creates the skeleton all three modules will plug into.

The frontend already exists at `frontend/` (React 19 + Vite). The backend will live at `backend/` at the same repo root level.

## User Story

As a backend developer on the Pigeon team, I want a fully scaffolded Express + TypeScript project with realistic seed data so that I can immediately begin implementing feature modules without setup friction.

## Acceptance Criteria

- [ ] `backend/` directory exists with valid `package.json`, `tsconfig.json` (strict mode), and `src/index.ts` entry point
- [ ] `npm run dev` starts the server on port 3000 without errors using `tsx`
- [ ] All shared TypeScript types are defined in `src/types/` and match the data models in the system design doc
- [ ] `src/data/shipments.json` contains 40 synthetic shipments with realistic port codes, carriers, SLA deadlines, legs, and purchase order links
- [ ] In-memory store singleton loads seed data on startup and exposes typed read/write operations
- [ ] `GET /health` returns `{ status: "ok", shipmentCount: 40 }`
- [ ] `GET /shipments` returns all 40 shipments
- [ ] `GET /shipments/:id` returns a single shipment or 404

## Technical Notes

- TypeScript strict mode + `noUncheckedIndexedAccess` + `exactOptionalPropertyTypes`
- Use `tsx` for dev execution (no build step in dev)
- Use `zod` for defining and validating the seed data shape at load time
- Express 5 (not 4) — note async error handling is built-in
- Path aliases: `@/types`, `@/store`, `@/data`, `@/modules` via tsconfig `paths`
- `.env.example` must document all 8 required environment variables (even if not used in this track)
- Seed data: use real UN/LOCODE port codes, real carrier names (Maersk, Hapag-Lloyd, Evergreen, MSC, CMA CGM), SLA deadlines 2–7 days from 2026-05-13, varied risk profiles (some green, some amber, some critical)
